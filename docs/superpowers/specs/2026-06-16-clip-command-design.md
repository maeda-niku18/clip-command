# clip-command 設計書

作成日: 2026-06-16

## 1. 概要

macOS 向けのクリップボード履歴 / スニペット管理アプリ。Clipy と同等の機能・ショートカットを踏襲しつつ、Clipy の弱点を構造的に解消する。メニューバー常駐のエージェントアプリとして動作し、App Store 配信（サンドボックス）を前提とする。

### 解決する Clipy の弱点

| # | Clipy の弱点 | 本アプリでの解決 |
|---|---|---|
| 1 | 最新 macOS / Apple Silicon で不安定・署名/起動の問題 | 最新 Swift + AppKit/SwiftUI でネイティブ実装。App Sandbox 対応 |
| 2 | 履歴の全文検索ができない | パネル上部に検索フィールド、インクリメンタル全文検索 |
| 3 | スニペット編集が面倒（別ウィンドウ・XMLベース） | アプリ内 SwiftUI エディタ。履歴パネル下部から即登録する軽量導線も用意 |
| 4 | 履歴が消える / プレビューが弱い | SwiftData で堅牢に永続化。画像は外部ファイル参照で保持 |
| 5 | メニューが狭く中身が見切れる | hover 中の行の横にワイドプレビューをポップ表示 |

## 2. 全体構成

- アプリ形態: メニューバー常駐エージェント（`LSUIElement = YES`、Dock 非表示、通常ウィンドウなし）
- ライフサイクル: AppKit（`NSApplicationDelegate`）を主軸に、UI 内部は SwiftUI を埋め込む（`NSHostingView` / `NSHostingController`）
- 配布: App Store。App Sandbox 有効。Apple Developer Program 加入が前提
- 言語/最低OS: Swift 最新、macOS の妥当な最低バージョン（実装着手時に確定。SwiftData 都合で macOS 14+ を想定）

### レイヤ分割（テスト容易性のため UI と副作用を分離）

| レイヤ | 責務 | 依存 |
|---|---|---|
| Core（ロジック） | 履歴管理・重複排除・上限制御・検索フィルタ・スニペット CRUD・並べ替え | OS API に非依存（純粋ロジック） |
| Persistence | SwiftData による履歴/スニペットの保存・復元、画像ファイルの読み書き | SwiftData / FileManager |
| Platform（薄いラッパ） | グローバルホットキー / クリップボード監視 / 自動貼り付け / アクセシビリティ権限 | Carbon / NSPasteboard / CGEvent / AX API |
| UI | 履歴パネル・スニペットパネル・スニペット編集・設定画面 | SwiftUI / AppKit |

Core はプロトコル経由で Persistence / Platform を受け取り、ユニットテスト時はフェイク実装に差し替える。

## 3. グローバルショートカット

Carbon `RegisterEventHotKey` で実装（サンドボックスで利用可）。すべて設定で変更可能。

| 機能 | デフォルト | 備考 |
|---|---|---|
| 履歴パネル表示 | ⌘⇧V | Clipy 準拠 |
| スニペットパネル表示 | ⌘⇧B | Clipy 準拠 |
| クリップボード内容を即スニペット登録 | ⌘⇧S | 補助導線（簡易命名ダイアログ） |

ホットキーは `HotKeyManager`（Platform 層）にカプセル化し、キーコンビ → アクションのマッピングを Core/設定から注入する。

## 4. クリップボード監視

- `NSPasteboard.general.changeCount` をタイマーでポーリング（間隔は設定可、デフォルト 0.5 秒程度）
- 変化検知時に内容を取り込む:
  - テキスト（`NSPasteboard.PasteboardType.string` / RTF があればプレーン化して保持、表示はプレーン）
  - 画像（`NSImage` 取得可能な場合）
- 取り込みルール:
  - 直近と同一内容なら重複として無視、または既存項目を先頭へ移動（重複排除）
  - 新規は履歴の先頭に追加
  - 件数が上限（デフォルト 50、設定可）を超えたら末尾から削除（画像の外部ファイルも削除）
- 監視対象から除外: 自アプリが貼り付けのために pasteboard を書き換えた直後の変化は取り込まない（フラグ制御）

`PasteboardMonitor`（Platform 層）が変化を Core に通知。取り込み判断・重複排除・上限制御は Core 側の純粋ロジック。

## 5. 永続化（SwiftData）

### モデル

- `ClipItem`
  - `id`, `createdAt`, `kind`(text/image)
  - `text`: テキスト本体（text の場合）
  - `imageFileName`: 画像ファイル名（image の場合。実体は Application Support 配下に保存）
  - `charCount` など表示用メタ（派生でも可）
- `SnippetFolder`
  - `id`, `name`, `order`, `snippets`(1対多)
- `Snippet`
  - `id`, `title`, `content`, `order`, 所属フォルダ

### 画像の扱い

- 画像本体は SwiftData に BLOB で持たず、`Application Support/clip-command/images/<uuid>.png` に保存し、モデルはファイル名のみ参照
- 履歴削除・上限超過時に対応ファイルも削除
- サンドボックスのコンテナ内に保存するため追加 entitlement 不要

### 起動時復元

- 起動時に SwiftData から履歴・スニペットを読み込みメモリに展開。画像はサムネ表示時に遅延ロード

## 6. 履歴パネル（メイン UI）

⌘⇧V で表示する自前のボーダーレス `NSPanel`（`.nonactivatingPanel` 系、フォーカスを奪いつつ前面アプリへ貼り付け可能にする）。内部は SwiftUI。

### レイアウト

```
┌─────────────────────────────────────────┐
│ 🔍 検索…                                  │  ← 上部: 検索フィールド
├─────────────────────────────────────────┤
│ ▸ 履歴1            ┌──────────────────┐  │
│   履歴2            │ hover でワイド    │  │  ← 中央: コンパクトな1行リスト
│   履歴3            │ プレビューがポップ │  │     hover/↑↓選択した行の横に
│   …               └──────────────────┘  │     ワイドプレビューをポップ
├─────────────────────────────────────────┤
│ ✎ ここに入力してスニペット登録…   [詳細▸]│  ← 下部: スニペット登録バー
└─────────────────────────────────────────┘
```

### 挙動

- 表示位置: マウスカーソル付近（画面端は内側に収める）
- 初期サイズは Clipy より広め。リサイズ可（サイズは記憶）
- 検索: 入力に応じてテキスト内容をインクリメンタルにフィルタ
- リスト: 各行は1行サマリ（テキスト先頭 / 画像はサムネ＋ラベル）
- ワイドプレビュー（弱点 5 の解消）:
  - hover または ↑↓ で選択した行の**横にポップ**（行にアンカーした popover / flyout）
  - テキスト: 全文を折り返し＋スクロール表示、文字数・コピー日時を表示
  - 画像: 大きく表示
- 確定: Enter またはクリックで選択項目を前面アプリへ自動貼り付け（§8）
- Esc でパネルを閉じる。キーボードのみで検索→選択→貼り付けが完結

### 下部スニペット登録バー（弱点 3 の軽量導線）

- テキストフィールドに直接入力 → Enter で即スニペット化（デフォルトフォルダへ）
- 現在 hover している履歴 / クリップボード内容をワンタップで流し込んで登録も可
- `[詳細▸]` でスニペット編集ウィンドウ（§7）を開く

## 7. スニペット機能

### パネル（⌘⇧B）

- 履歴パネルと同様のフローティングパネル
- フォルダ → スニペットの2階層を辿り、選択で前面アプリへ自動貼り付け

### 編集ウィンドウ（SwiftUI、通常ウィンドウ）

- フォルダの追加・リネーム・並べ替え・削除
- スニペットの追加・編集（タイトル / 本文）・並べ替え・フォルダ間移動・削除
- Clipy の XML 編集を廃し、アプリ内で完結

### 即登録

- ⌘⇧S: 現在のクリップボード内容を簡易命名ダイアログでスニペット登録
- 履歴パネル下部バー: §6 参照

## 8. 自動貼り付け

- 流れ: 選択項目を `NSPasteboard.general` にセット → 監視側に「自分の書き込み」フラグ → `CGEvent` で ⌘V を前面アプリに送出
- 権限: `AXIsProcessTrusted()` でアクセシビリティ許可を確認。未許可ならシステム設定の該当ペインへ誘導するガイドを表示
- フォールバック: 許可が無い場合はクリップボードにセットのみ行い、ユーザーに手動 ⌘V を促す（パネルは閉じる）
- パネルは `.nonactivatingPanel` とし、前面アプリのフォーカスを維持したまま操作する

## 9. 設定画面（SwiftUI）

- 履歴保持件数（デフォルト 50、スライダ/数値入力）
- 各グローバルショートカットの変更（キーキャプチャ UI）
- 貼り付け方式（自動貼り付け / クリップボードにセットのみ）
- 監視間隔
- ログイン時に自動起動（`SMAppService`）
- 履歴のクリア、画像保存容量の表示

## 10. テスト方針

UI 非依存の Core ロジックを中心にユニットテスト:

- 履歴: 追加 / 重複排除 / 先頭移動 / 上限超過削除（画像ファイル削除含む）
- 検索フィルタ: クエリに対する一致挙動
- スニペット CRUD: フォルダ/スニペットの追加・並べ替え・移動・削除
- 永続化: 保存→復元のラウンドトリップ（画像ファイル参照の整合）

Platform 層（ホットキー / 監視 / 貼り付け / AX 権限）はプロトコルでフェイク化し、Core のテストから分離。実機依存部分は手動確認とする。

## 11. App Store / サンドボックス上の前提

- App Sandbox 有効。画像はコンテナ内に保存（追加 entitlement 不要）
- グローバルホットキー（Carbon）はサンドボックスで動作
- 自動貼り付けはユーザーがアクセシビリティ権限を付与した場合のみ有効（実行時許可）
- Apple Developer Program 加入・Developer ID/配信用証明書が別途必要
- App Review 上、クリップボード管理 + アクセシビリティ利用の正当性を説明できるようにしておく（先例: Paste, Pastebot）

## 12. スコープ外（YAGNI）

- クラウド同期 / 複数デバイス共有
- 履歴のファイル（任意バイナリ）対応 ※テキスト＋画像のみ
- リッチテキスト書式の完全保持（プレーン化して保持）
- iCloud / iOS 版
